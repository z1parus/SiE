# Спецификация системы AI-декомпозиции целей (GROQ/Llama)

## 1. Обзор функционала
Цель — предоставить пользователю "Интеллектуального помощника", который автоматически разбивает Глобальную Цель (Миссию) на структурированный план, состоящий из Этапов (Operations), Задач с весами (Tasks) и Контрольных точек (Milestones).

В качестве провайдера используется **GROQ API**, модель — **Llama 3** (8b или 70b).

---

## 2. Техническая архитектура

### А. Сервисный слой (`sie_core`)
Создание нового сервиса `GroqAiService` для взаимодействия с API GROQ.

*   **Endpoint:** `https://api.groq.com/openai/v1/chat/completions` (совместим с OpenAI SDK).
*   **Авторизация:** API Ключ (через переменные окружения или защищенное хранилище).
*   **Основной метод:** `Future<DecompositionResult> decomposeGoal(String name, String? description)`

### Б. Промпт-инжиниринг (Системные инструкции)
Системный промпт определяет правила декомпозиции:
*   **Роль:** Эксперт по стратегическому планированию в экосистеме SiE (System in Evolution).
*   **Структура вывода:** 
    *   **Этапы (Sub-goals):** 3-5 логических фаз.
    *   **Задачи (Tasks):** 3-7 конкретных действий на каждый этап.
    *   **Веса (Weights):** Присвоение веса `1` (легкая), `3` (средняя) или `5` (сложная) в зависимости от усилий.
    *   **Чекпоинты (Milestones):** 2-4 ключевые точки контроля.
*   **Формат:** СТРОГО JSON.

#### Пример системного промпта:
> "Ты — AI-стратег системы SiE. Разбей цель пользователя на детальный план. 
> Выведи ТОЛЬКО JSON-объект следующей структуры:
> {
>   'sub_goals': [
>     {
>       'name': 'название этапа',
>       'tasks': [{'name': 'текст задачи', 'weight': 1|3|5}]
>     }
>   ],
>   'milestones': [{'name': 'название точки'}]
> }
> Используй вес 1 для простых действий, 3 для сфокусированной работы и 5 для крупных задач."

## 3. Модели и сопоставление данных

### А. Модель результата AI
```dart
class DecompositionResult {
  final List<AiSubGoal> subGoals;
  final List<AiMilestone> milestones;
}
```

### Б. Маппинг на существующую схему
*   `AiSubGoal` -> Таблица `sub_goals` (поле `goal_id`).
*   `AiTask` -> Таблица `planning_tasks` (связь с `sub_goal_id`).
*   `AiMilestone` -> Таблица `milestones` (поле `goal_id`).

## 4. Пользовательский сценарий (UX)

### Этап 1: Инициация
*   На экране `MissionDetailScreen` добавляется кнопка "Декомпозировать (AI)".
*   В процессе создания новой цели предлагается опция "Сгенерировать план".

### Этап 2: Генерация
*   Полноэкранный оверлей "Тактическое планирование AI".
*   Анимация процесса (например, пульсирующая сфера или симуляция терминала).

### Этап 3: Проверка и принятие
*   Отображение сгенерированного списка.
*   Возможности пользователя:
    *   **Принять:** Пакетная вставка всех элементов в БД.
    *   **Редактировать:** Возможность изменить веса или названия перед сохранением.
    *   **Перегенерировать:** Запрос альтернативного варианта.

---

## 5. Безопасность и лимиты
*   **Защита ключа:** Ключ GROQ_API_KEY не коммитится в репозиторий.
*   **Лимиты:** Клиентский debounce (задержка) и ограничение количества запросов в день для предотвращения лишних затрат по API.
*   **Приватность:** Уведомление пользователя, что название цели отправляется на внешний сервис.
