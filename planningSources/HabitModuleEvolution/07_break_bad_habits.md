# Этап 7 — Отказ от вредных привычек

## Описание

Текущая модель привычки — только **«делать полезное»** (выполнил → стрик растёт). Но огромный
запрос пользователей — **«перестать делать вредное»**: бросить курить, не пить алкоголь, не
заходить в соцсети, не есть сахар. Это **принципиально другая ментальная модель**: успех = «день
прошёл БЕЗ срыва», а ценность — в **счётчике дней воздержания** («14 дней без сигарет») и в
понимании, что срыв стоит. Этот этап добавляет тип привычки «**избегать**» (negative/avoidance) с
зеркальной логикой: воздержание = успех, срыв = обнуление счётчика (с поддерживающей подачей).

## Пользовательский сценарий

1. При создании привычки выбирается полярность: **«Развивать»** (как сейчас) или **«Избавиться»**.
2. Для «Избавиться» главная метрика — **«N дней без …»** крупным числом (счётчик воздержания от
   последнего срыва). Карточка визуально иная (например, «щит» вместо «огня»).
3. Каждый прожитый день засчитывается автоматически как успех воздержания (или подтверждается
   пользователем — см. открытые вопросы); кнопка на карточке — **«Был срыв»** (не «выполнить»).
4. При нажатии «Был срыв» — счётчик обнуляется, фиксируется дата срыва; поддерживающий тон
   («Срыв — часть пути. Твой рекорд 21 день. Продолжаем.»). Опционально — заметка/эмодзи (журнал,
   уже есть) с триггером срыва для рефлексии.
5. Аналитика (этап 4): частота срывов, самый долгий период чистоты, тренд («срывы реже»).

## Логика и поведение

### Модель
- `Habit.polarity: String` — `'build'` (default, текущее) | `'avoid'`.
- Для `avoid` лог имеет смысл «срыв»: переиспользуем `habit_logs`, но `entry_type='lapse'`
  (срыв) вместо `'done'`. День без лога-срыва = успешный день воздержания.
- Счётчик воздержания = дней с последнего `lapse` (или с `createdAt`, если срывов не было).

### Зеркальные стрик/аналитика
- `abstinenceStreak(habit, lapses)` = `today - lastLapseDate` (в днях).
- `longestClean(habit, lapses)` — максимальный интервал между срывами.
- Здесь стрик **растёт сам по себе** с течением времени (не требует ежедневного действия) — поэтому
  расписание (этап 1) для `avoid` обычно `daily`/не применяется; фокус на непрерывности.

### Геймификация (осторожно)
- XP за `avoid`: НЕ начислять «за каждый день» автоматически (это фарм времени без действия).
  Рекомендация: начислять **майлстоун-XP за рубежи воздержания** (1/7/30/90 дней без срыва) —
  событийно, через достижения; согласуется с существующей системой `achievements`/медалей.
- Срыв XP не отнимает (не наказываем), только обнуляет счётчик.

### Edge cases
- **Авто-успех vs подтверждение** — считать день воздержания автоматически (нет действия = успех)
  или требовать ежедневного «чек-ина чистоты»? Рекомендация: **авто** (меньше трения; «избегание»
  по природе пассивно), с возможностью отметить срыв в любой момент, в т.ч. задним числом (ограничив
  разумным окном).
- **Несколько срывов в день** — один `lapse`-лог на дату (UNIQUE сохраняется); счётчик всё равно
  обнуляется.
- **Habit Synergy** — `avoid`-привычки обычно не «питают» прогресс цели бустом действия; если
  линкуются с целью («бросить курить» → цель «здоровье»), буст давать за рубежи воздержания, не
  ежедневно.
- **Обратная совместимость** — все текущие привычки `polarity='build'`.

### Схема данных

```sql
ALTER TABLE public.habits
  ADD COLUMN IF NOT EXISTS polarity text NOT NULL DEFAULT 'build';  -- 'build'|'avoid'
-- habit_logs.entry_type расширяется значением 'lapse' (срыв) — таблица из этапа 5
```

## Открытые вопросы

1. **Авто-успех или ежедневный чек-ин чистоты?** Рекомендация: авто (минимум трения), срыв —
   явным действием.
2. **XP за воздержание** — событийные рубежи (рекоменд.) или ничего/ежедневно? Рекомендация:
   рубежи через `achievements` (1/7/30/90 дней) — мотивирует без фарма времени.
3. **Отдельный визуальный язык** для `avoid` (щит/чистота vs огонь/рост)? Рекомендация: да — разная
   ментальная модель требует разного UI-акцента, иначе путаница «выполнить vs срыв».
