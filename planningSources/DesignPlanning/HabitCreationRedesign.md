# Спецификация редизайна интерфейса "Создание привычки"

## 1. Цель
Унифицировать дизайн модальных окон создания привычек и рутин с интерфейсом "PLANNING DEPARTAMENT". 

## 2. Визуальные изменения (Модальное окно)

### 2.1. Контейнер и позиционирование
- **Отступы (Margin):** Добавить внешние отступы `EdgeInsets.fromLTRB(12, 0, 12, 16)`, чтобы окно не примыкало к краям экрана.
- **Скругление:** Использовать `BorderRadius.circular(16)` для всех углов (сейчас только верхние).
- **Фон:** 
    - Убрать динамическую смену цвета фона при выборе цвета привычки.
    - Использовать статичный `sc.surface` с сохранением эффекта `BackdropFilter` (Blur 35).
    - Граница (Border) должна быть константной `sc.border`.

### 2.2. Типографика
- Привести заголовки секций ("TITLE", "COLOR", "ICON") к стилю Planning:
    - `fontSize: 10`
    - `fontWeight: FontWeight.w700`
    - `letterSpacing: 2.5`
    - Цвет: `sc.textSecondary`
    - Регистр: ВЕРХНИЙ (All Caps).

### 2.3. Селектор цвета
- Заменить текущий `_ColorLens` на упрощенный стиль из Planning:
    - Размер кружков: 32x32.
    - Отсутствие теней (Shadows) при выборе.
    - Индикация выбора: внутренняя белая обводка `Border.all(color: Colors.white, width: 2.5)`.
    - Палитра цветов должна быть расширена до стандарта Planning или объединена.

## 3. Технические детали реализации

### 3.1. Виджет `_HabitDialog` (habit_tracker_screen.dart)
- Удалить `TweenAnimationBuilder<Color?>`.
- Обернуть основной контент в `Container` с указанными выше марджинами.
- Настроить `SafeArea` или `MediaQuery.viewInsetsOf(context).bottom` для корректного отображения над клавиатурой.

### 3.2. Унификация для рутин
- **_HabitPickerSheet (routine_editor_screen.dart):**
    - Обернуть в такой же "парящий" контейнер с `margin: 12` и `borderRadius: 16`.
    - Убрать привязку к верху экрана (сейчас `borderRadius.vertical(top: ...)`).
    - Привести заголовки к стандарту Planning.

## 4. Ожидаемый результат
Все модальные окна, связанные с привычками (создание, выбор для рутины), выглядят как "парящие" карточки, визуально идентичные окну "НОВАЯ МИССИЯ" из раздела планирования. Это создаёт ощущение единой, модульной системы интерфейса.
