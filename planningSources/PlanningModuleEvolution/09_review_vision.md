# Этап 9 — Ритуал ревью + слой смысла (Vision)

## Описание

Капстоун-этап, связывающий всё предыдущее в **привычку рефлексии** и **мотивационный якорь**.
Состоит из двух взаимодополняющих частей:

1. **Еженедельный ревью-ритуал** — направляемый еженедельный обзор: что выполнено, где застрял,
   что перенести, на чём сфокусироваться на следующей неделе. Концепт-документ модуля прямо называл
   «Ретроспективу» как нереализованную функцию. Ритуал — мощнейший механизм **удержания** и
   реального доведения целей: он возвращает пользователя «над картой», а не только «в задачах».
2. **Слой смысла (Vision / «Зачем»)** — у цели появляется «Зачем я это делаю» (мотивация/ценность),
   и опционально общий «Vision board» жизненных направлений. Это поднимает планирование с уровня
   «список дел» на уровень «осмысленная траектория», и подпитывает мотивацию в момент усталости.

## Пользовательский сценарий

### Ревью
1. Раз в неделю (настраиваемый день, напр. воскресенье вечером) приходит приглашение «🧭 Время
   еженедельного обзора» (через систему уведомлений, этап 2).
2. Открывается направляемый флоу (несколько карточек-шагов):
   - **Итоги недели:** «Выполнено 12 задач, 1 веха, +1 уровень. Лучший день — среда.» (данные из
     аналитики этапа 5 и логов).
   - **Застрявшее:** список целей с падающим импульсом (`momentumState=stalling/atRisk`) + кнопки
     «Заморозить / Разбить / Перенести дедлайн».
   - **Просроченное:** пакетный перенос просроченных задач (из War Room) — «перенести всё на эту
     неделю».
   - **Фокус недели:** выбрать 1–3 цели/задачи как приоритет недели (помечаются ⭐ в War Room).
   - **Рефлексия:** короткое текстовое поле «Главный вывод недели» (сохраняется в журнал ревью).
3. Завершение ревью — небольшая награда (XP/DP + «Zen-подобный» стрик ревью, как у медитации) и
   позитивный экран-итог.

### Vision
1. У цели в настройках — поле «**Зачем** (моя мотивация)». Отображается на видном месте в
   `MissionDetailScreen` (мягкая цитата под заголовком) — напоминание о смысле при работе.
2. (Опц.) Экран «**Видение**» — список жизненных направлений/ценностей; цели можно привязывать к
   направлению, видеть баланс («в карьеру вложено 60% усилий, в здоровье — 10%»).

## Логика и поведение

### Ревью
- Данные ритуала **агрегируются из уже существующего** (этапы 1, 5): задачи за неделю, медали,
  XP-дельта, импульс целей. Новое — только журнал ревью и пометки «фокус недели».
- Таблица `weekly_reviews`: `id`, `userId`, `weekStart`, `completedTasks`, `notes`, `focusGoalIds`
  (jsonb), `createdAt`. Плюс счётчик `review_streak` (по аналогии с `zen_streak_days` медитации —
  тот же паттерн в `profiles`).
- «Фокус недели» — пометка целей/задач: либо поле `is_weekly_focus`/срок, либо хранится в
  `weekly_reviews.focusGoalIds` и читается War Room для ⭐.
- Триггер ритуала — уведомление этапа 2 (еженедельное), + ненавязчивый баннер «Доступен обзор недели»
  на экране планирования, если ревью за эту неделю не сделан.

### Vision
- Поле `why: String?` (мотивация) на `Goal`/`GoalSettings` — простое текстовое расширение.
- (Опц.) Сущность `life_areas` (направления) + связь `goal.lifeAreaId` — для баланса усилий.
  Можно отложить; «Зачем» на цели — минимально ценная часть, внедряется первой.

### Edge cases
- **Пропущенная неделя** — стрик ревью сбрасывается (как zen-streak); не ругать, мягко предложить
  «начать заново».
- **Нет активности за неделю** — ритуал всё равно полезен («тихая неделя — норм, перезагрузимся»),
  не показывать пустой укоряющий экран.
- **Часовые пояса / граница недели** — `weekStart` считать по локальной зоне, ISO-неделя (Пн–Вс
  или настраиваемо).
- **Антифарм** — награда за ревью даётся раз в неделю (по `weekStart` уникальность), нельзя
  «прокликать» несколько ревью за день.

## Схема данных

```sql
ALTER TABLE public.goals     ADD COLUMN IF NOT EXISTS why text;
ALTER TABLE public.profiles  ADD COLUMN IF NOT EXISTS review_streak int NOT NULL DEFAULT 0;

CREATE TABLE public.weekly_reviews (
  id              uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id         uuid NOT NULL REFERENCES public.profiles(id) ON DELETE CASCADE,
  week_start      date NOT NULL,
  completed_tasks int  NOT NULL DEFAULT 0,
  notes           text,
  focus_goal_ids  jsonb NOT NULL DEFAULT '[]',
  created_at      timestamptz NOT NULL DEFAULT now(),
  UNIQUE (user_id, week_start)     -- 1 ревью на неделю (антифарм)
);
-- RLS: owner-only select/insert
-- RPC (опц.) log_weekly_review(...) — по образцу log_meditation_session: пишет ревью,
--   начисляет XP/DP, обновляет review_streak (та же логика стрика, что zen_streak_days)

-- (опц.) направления жизни
CREATE TABLE public.life_areas (
  id      uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id uuid NOT NULL REFERENCES public.profiles(id) ON DELETE CASCADE,
  name    text NOT NULL,
  color   text
);
-- goals.life_area_id uuid REFERENCES life_areas(id) ON DELETE SET NULL
```

## Открытые вопросы

1. **Объём ревью-флоу** — полный (5 шагов) или лёгкий (итоги + фокус недели)? Рекомендация: начать
   с лёгкого (итоги + перенос просроченного + фокус недели), рефлексию-текст добавить вторым шагом.
2. **Vision: только поле «Зачем» или полноценный Vision board с направлениями?** Рекомендация:
   MVP — поле «Зачем» на цели (огромная ценность за копейки). `life_areas`/баланс — отдельный
   подэтап, если зайдёт.
3. **Награда за ревью** — XP/DP и/или стрик ревью как у медитации? Рекомендация: переиспользовать
   паттерн `zen_streak_days` (RPC `log_weekly_review`) — консистентно с существующей геймификацией.
4. **День/время ритуала** — фиксированное воскресенье или настраиваемое? Рекомендация: дефолт
   воскресенье 19:00, настраиваемо (связано с настройками уведомлений этапа 2).
