# Этап 6 — Шаблоны миссий (Blueprints)

## Описание

Каждая цель сейчас создаётся «с чистого листа». Это высокий порог входа: пользователь, который
хочет «выучить язык» или «подготовиться к марафону», должен сам придумать всю структуру под-целей
и задач. Этап вводит **шаблоны миссий** — готовые чертежи структуры (под-цели + задачи + вехи +
категория + связи с привычками), которые разворачиваются в полноценную цель одним тапом.

Два источника шаблонов:
1. **Системные** (курируемые) — «Выучить язык», «Запустить пет-проект», «Подготовка к марафону»,
   «Финансовая подушка» и т.д.
2. **Пользовательские** — «Сохранить как шаблон» из любой существующей цели, чтобы переиспользовать
   свою проверенную структуру (или поделиться — в будущем).

Паттерн архитектурно повторяет уже реализованные **пресеты медитации** (`meditation_presets`),
так что в кодовой базе есть готовый образец.

## Пользовательский сценарий

1. На экране планирования кнопка **«+»** предлагает: «Пустая цель» / «Из шаблона» / «AI-декомпозиция».
2. «Из шаблона» → галерея карточек (системные + мои), сгруппированы по категориям, с превью
   («5 этапов · 18 задач · 3 вехи»).
3. Тап по шаблону → экран предпросмотра структуры → «Создать миссию» (можно отредактировать
   название/дедлайн перед созданием).
4. Цель создаётся со всей структурой; пользователь сразу попадает в `MissionDetailScreen`.
5. Из меню любой существующей цели → «**Сохранить как шаблон**» → структура (без личных дат и
   прогресса) сохраняется в «Мои шаблоны».

## Логика и поведение

### Хранение шаблона
- Шаблон — это **сериализованная структура** (без прогресса/дат/ID конкретных экземпляров):
  дерево под-целей с задачами (имя+вес), список вех (имена/типы), категория, цвет, предлагаемые
  связи с типами привычек.
- Формат — JSON-документ (как `DecompositionResult`, но сохраняемый). Это даёт гибкость без
  раздувания реляционной схемы.
- Таблица `mission_templates`: `id`, `userId?` (NULL для системных), `name`, `description`,
  `category`, `isSystem`, `structureJson` (jsonb), `createdAt`.

### Разворачивание (instantiate)
- Новый метод `createGoalFromTemplate(template, {name, deadline})` в `PlanningNotifier`:
  - создать `Goal` (новый id, выбранные имя/дедлайн/категория/цвет);
  - рекурсивно создать под-цели и задачи из `structureJson` (новые id, `orderIndex` по порядку);
  - создать вехи;
  - всё — через существующие batch-вставки в Drift + sync-очередь.
- Переиспользовать логику массовой вставки из **`applyAiDecomposition`** — она уже умеет разворачивать
  дерево под-целей+задач. По сути шаблон = заранее заготовленный `DecompositionResult` + метаданные.

### Сохранение из цели
- `saveGoalAsTemplate(goalId, name)` — обойти дерево цели, собрать структуру (имена/веса/типы),
  обнулить личные данные (даты, прогресс, completed-флаги), записать как пользовательский шаблон.

### Edge cases
- **Глубоко вложенные деревья** — сериализация рекурсивна, лимит глубины (напр. 4) для вменяемости.
- **Связи с привычками** — в шаблоне хранить не конкретный `habitId` (он у каждого свой), а
  «предложение создать/связать привычку типа X»; при разворачивании показать подсказку, но не
  связывать автоматически.
- **Системные шаблоны** — `isSystem=true`, `userId=NULL`, доступны всем (RLS select), редактировать
  нельзя (только «дублировать в мои»).
- **Офлайн** — пользовательские шаблоны работают локально (Drift), синк при появлении сети.

## Схема данных

```sql
CREATE TABLE public.mission_templates (
  id            uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id       uuid REFERENCES public.profiles(id) ON DELETE CASCADE,  -- NULL = системный
  name          text NOT NULL,
  description   text,
  category      text,
  is_system     boolean NOT NULL DEFAULT false,
  structure_json jsonb NOT NULL,   -- {subGoals:[{name,tasks:[{name,weight}],children:[...]}], milestones:[...]}
  created_at    timestamptz NOT NULL DEFAULT now()
);
-- RLS select: user_id = auth.uid() OR is_system = true
-- RLS insert/update/delete: owner-only AND is_system = false

-- сид (пример)
INSERT INTO mission_templates (name, category, is_system, structure_json) VALUES
  ('Выучить язык',        'learning', true, '{...}'),
  ('Запустить пет-проект','project',  true, '{...}'),
  ('Подготовка к марафону','health',  true, '{...}'),
  ('Финансовая подушка',  'project',  true, '{...}');
```

## Открытые вопросы

1. **Делиться пользовательскими шаблонами** (публичная библиотека сообщества)? Мощно для роста,
   но требует модерации. Рекомендация: вне scope MVP; заложить поле `is_public` на будущее.
2. **Связь с AI-декомпозицией** — предлагать «сохранить AI-результат как шаблон»? Логично и дёшево
   (структура уже в нужном формате). Рекомендация: да, как быстрый бонус.
3. **Стартовый набор системных шаблонов** — какие 6–8 включить? Предложение: язык, пет-проект,
   марафон/похудение, накопления, чтение книг (N в год), смена работы, ремонт. Финальный список —
   за продуктом.
