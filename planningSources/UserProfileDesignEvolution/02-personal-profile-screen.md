# Этап 2 — Экран личного профиля

> **Статус: реализовано (2026-06-14).** Сделано: вход в «Облик» 3-й плиткой
> быстрых действий (Progress / База знаний / Облик); верхняя карточка — общий
> `ProfileHeroCard` (этап 1); полоса статов `_StatsStrip` (Уровень / XP всего /
> DP) со стилем статов вместо одинокого баннера; скелетоны (`SieSkeletonGrid`)
> вместо спиннеров в сетках ачивок и медалей; счётчик «получено / всего» в
> заголовке AWARDS. Проверено `flutter analyze` (без ошибок/варнингов).
> Не делалось: отдельный tap-feedback по бейджам ачивок (бейджи без onTap по
> текущему дизайну).

## Описание

Редизайн экрана `profile_screen.dart` (1043 строки) поверх нового
`ProfileHeroCard` (этап 1). Сейчас контент идёт сплошной лентой:
hero → карточка стиля статов → 2 квадратные навигационные плитки → сетка ачивок
→ хранилище медалей. Иерархия слабая, секции визуально не разделены, есть
P1/P2-замечания из `UX-Audit/05-social-profile.md §5.2`.

## Пользовательский сценарий

Свой профиль — это «личный кабинет оперативника». Сверху — премиальный hero с
фоном и узором. Ниже — компактная строка **ключевых показателей** (уровень,
суммарный XP, DP, дни-стрик), затем понятные **быстрые действия** (Прогресс,
База знаний, Облик), затем **достижения** и **витрина медалей**. Пользователь
сразу видит свой прогресс и легко переходит к кастомизации.

## Логика и поведение

### Информационная архитектура (сверху вниз)

1. **TopBar** (есть, 60–114): заголовок + кнопки «друзья» и «редактирование»
   (`_GlassCircleButton`). Добавить кнопку **«Облик»** (вход в Customization)
   рядом — сейчас попасть в кастомизацию из профиля неочевидно.
2. **ProfileHeroCard** (этап 1).
3. **Полоса статов (NEW `_StatsStrip`)** — 3–4 компактные ячейки в ряд:
   `Уровень`, `XP всего`, `DP`, `Стрик`. Заменяет/дополняет одинокую
   `_StatStyleCard` (510–552) — стиль статов применяется как оформление этой
   полосы (`buildStatCardDecoration`), а не отдельной карточки. Тач-таргеты и
   подписи ≥11px (UX-Audit 0.7).
4. **Быстрые действия** — `_SquareNavButton` (557–607) расширить с 2 до 3 плиток:
   `Прогресс` (Progress Hub), `База знаний`, `Облик`. Либо горизонтальная лента
   плиток с иконкой+подписью.
5. **Достижения** — `_AchievementsGrid` (612–653, 3 колонки) + заголовок
   `SectionHeader`. Добавить счётчик «N из M», тап по ачивке с
   feedback-анимацией и `SieHaptics.selection()` (UX-Audit 5.2 #4).
6. **Витрина медалей** — `_MedalsVault` (658–850). Sheet'ы `_MedalDetailSheet` /
   `_MedalGroupSheet` дополнить grabber'ом/тенью скролл-хинта (UX-Audit 5.2 #5).

### Состояния загрузки и пустоты

- Заменить центральные спиннеры на **`SieSkeleton`** (уже есть в `sie_core`):
  скелетон hero, полосы статов, сеток ачивок/медалей (UX-Audit 5.2 #3, 0.3).
- Пустые состояния ачивок/медалей — `SieEmptyState` с CTA «Как получить?»
  (UX-Audit 0.4): иконка + текст + ссылка на объяснение прогрессии.

### Микро-полировка (из аудита)

- Чипы 10px → 12px (закрыто в этапе 1, hero).
- XP-бар 6px → 10px (закрыто в этапе 1).
- Тап-feedback на ачивках/медалях: scale-down + хаптика.
- Группировка медалей — добавить понятные ярлыки групп (UX-Audit 5.2 #6).

### Визуальный ритм

- Единый горизонтальный padding (20), вертикальные отступы между секциями 24.
- Каждая секция предваряется `SectionHeader` (общий виджет
  `section_header.dart`) для единообразия с остальными экранами.
