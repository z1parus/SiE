 **Микро-заметки и Рефлексия (Habit Journaling):** Возможность прикрепить короткую текстовую заметку или эмодзи настроения к конкретному выполнению (например, зафиксировать вес после тренировки или мысли после медитации).

## 📝 Технический план реализации: Habit Journaling (Микро-заметки и Рефлексия)

**Описание фичи:** Пользователь может прикрепить короткую текстовую заметку и/або эмодзи к конкретному факту выполнения привычки за определенный день.

### 1. Изменения в БД (Supabase)
Необходимо создать новую миграцию (например, `20260606000001_habit_journaling.sql`), которая добавит новые столбцы в таблицу `habit_logs`:
```sql
ALTER TABLE public.habit_logs 
ADD COLUMN note TEXT,
ADD COLUMN emoji VARCHAR(10);
```

### 2. Изменения в локальной БД (Drift)
В файле `packages/sie_core/lib/src/local/app_database.dart` необходимо обновить таблицу `LocalHabitLogs`:
```dart
@DataClassName('LocalHabitLog')
class LocalHabitLogs extends Table {
  TextColumn get habitId => text()();
  TextColumn get userId => text()();
  TextColumn get completedAt => text()();
  TextColumn get note => text().nullable()(); // Новое поле
  TextColumn get emoji => text().nullable()(); // Новое поле
  BoolColumn get synced => boolean().withDefault(const Constant(false))();

  @override
  Set<Column> get primaryKey => {habitId, userId, completedAt};
}
```
*После изменения нужно запустить генерацию кода: `dart run build_runner build -d`.*

### 3. Изменение Data Models
На данный момент выполненные привычки хранятся в провайдере как `Map<String, Set<String>>` (где ключ - `habitId`, а значение - `Set` дат). 
Придется создать новый класс:
```dart
class HabitLogEntry {
  final String dateStr;
  final String? note;
  final String? emoji;
  
  HabitLogEntry({required this.dateStr, this.note, this.emoji});
}
```
И изменить стейт провайдера на `Map<String, Map<String, HabitLogEntry>>` (где вложенная мапа имеет ключ `dateStr`).

### 4. Изменения в Riverpod (habits_provider.dart)
* Метод `toggleHabit` (или новый метод `addHabitLog`) должен принимать опциональные параметры `String? note` и `String? emoji`.
* При записи в локальную БД и отправке в Supabase (в `insert` и `update`) нужно передавать эти новые поля.
* Офлайн-синхронизация (`sync_queue`) также должна сериализовать и отправлять эти поля.

### 5. Изменения в UI (HabitTrackerScreen и HabitCard)
* **Ввод:** По одиночному тапу привычка по-прежнему просто выполняется. При долгом нажатии (Long Press) открывается Bottom Sheet (или кастомный диалог `GlassCard`), где пользователь может выбрать эмодзи из шорт-листа и написать небольшой текст.
* **Альтернативный ввод:** После чек-ина (выполнения) под карточкой привычки может на 5 секунд появляться небольшая кнопка "+ Добавить рефлексию".
* **Отображение:** Если к привычке за сегодня прикреплен эмодзи, он отображается поверх или рядом с иконкой галочки в `HabitCard`.
* **История:** В детальном просмотре привычки нужно добавить список (Timeline) предыдущих выполнений, где будут выводиться даты и прикрепленные к ним заметки.

### 6. Пользовательский опыт (UX Flow)
Полный сценарий взаимодействия пользователя с новой функцией "Habit Journaling":

**Сценарий А: Быстрое выполнение (Без рефлексии)**
1. Пользователь видит карточку невыполненной привычки (например, "Чтение 20 страниц").
2. Делает обычный короткий тап (Tap) по карточке.
3. Проигрывается стандартная анимация (карточка загорается акцентным цветом, появляется галочка). Привычка отмечается как выполненная. Никаких дополнительных окон не всплывает, чтобы не ломать быстрый flow.
4. *Новая деталь:* Сразу после короткого тапа под карточкой на 3-5 секунд плавно появляется небольшая кнопка или текст: `+ Добавить заметку (Long Press)`. Затем она исчезает, чтобы не перегружать интерфейс.

**Сценарий Б: Выполнение с рефлексией (Long Press)**
1. Пользователь выполнил сложную привычку (например, "Тяжелая тренировка") и хочет зафиксировать вес или ощущения.
2. Он делает долгое нажатие (Long Press) на карточку привычки.
3. Поверх экрана плавно (с блюром на фоне) выезжает модальное окно (Bottom Sheet / Glass Card) с заголовком: *"Тяжелая тренировка: Рефлексия"*.
4. **Интерфейс модального окна:**
   - Карусель или сетка из 5-7 базовых эмодзи настроения (🔥, 🧘‍♂️, 😴, 🎯, 💧, 📈, 📉).
   - Текстовое поле (TextField) с плейсхолдером: *"Как все прошло? Оставьте заметку..."* (ограничение, например, 150 символов).
   - Кнопка "Сохранить".
5. Пользователь выбирает эмодзи (например, 🔥), пишет "Сделал становую 100кг!" и нажимает "Сохранить".
6. Модальное окно закрывается. Привычка автоматически отмечается как выполненная (если не была выполнена ранее).
7. **Визуальный фидбек:** На карточке привычки, рядом со стандартной галочкой выполнения (или вместо нее), появляется выбранный эмодзи (🔥). Это дает мгновенное понимание, что день прошел не просто "выполнено", а с особым статусом.

**Сценарий В: Редактирование уже выполненной привычки**
1. Пользователь уже отметил привычку выполненной утром (коротким тапом).
2. Вечером он решает добавить к ней мысль. Он делает Long Press по уже выполненной (подсвеченной) карточке.
3. Открывается то же модальное окно, но уже с кнопкой "Обновить" (и возможностью удалить заметку).
4. *Примечание:* Короткий тап по уже выполненной карточке по-прежнему должен просто снимать отметку выполнения (снимать галочку), с предупреждением: "Удалить запись о выполнении и заметку?".

**Сценарий Г: Просмотр истории (Timeline)**
1. Пользователь нажимает на иконку "Инфо" или делает свайп для перехода на экран деталей конкретной привычки (`HabitDetailScreen`).
2. Под графиком стриков появляется новый блок: **"Журнал (Journal)"**.
3. В этом блоке в виде красивого таймлайна (вертикального списка с датами слева) выводятся все дни, когда к выполнению была прикреплена заметка или эмодзи.
4. *UX профит:* Пользователь может пролистать историю и увидеть, как менялись его ощущения от тренировок или медитаций месяц назад, что значительно повышает ценность приложения как личного дневника.
