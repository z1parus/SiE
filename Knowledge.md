# SiE: Self-improvement Ecosystem — Knowledge Ledger

## 1. Концепция и Философия
SiE — это децентрализованная экосистема микро-приложений для саморазвития. 
- **Основная идея**: Вместо одного тяжелого приложения — сеть легких «филиалов».
- **Игровая метафора**: Пользователь — «Оперативник», инструменты — «Департаменты/Филиалы», развитие — «Прокачка корпорации».
- **Монетизация**: Единая подписка $1 за доступ ко всей экосистеме.

## 2. Технологический стек
- **Frontend**: Flutter (Monorepo).
- **Backend**: Supabase (Postgres, Auth, Realtime).
- **State Management**: Riverpod.
- **Инфраструктура**: Docker (локальный запуск бэкенда через Supabase CLI).

## 3. Архитектура хранилища (Monorepo)
- `apps/central_hub`: Главная панель управления, профиль, авторизация.
- `packages/sie_core`: Общая логика, UI-компоненты, темы, сервисы связи с API.
- `apps/[micro-app-name]`: (в планах) Отдельные репозитории или папки для инструментов.

## 4. Текущее состояние (Checkpoint)
### Реализовано:
- **Auth**: Регистрация/Вход через Supabase Auth + Mailpit для локальных тестов.
- **Profiles**: Автоматическое создание профиля через SQL-триггер при регистрации.
- **Branch: Breathing Practices**:
    - Метод Вима Хофа (Вдох, Выдох, Задержка на выдохе, Задержка на вдохе).
    - Динамическая подстройка скорости аудио под настройки сессии.
    - Система частичного начисления XP (10 XP/мин) даже при досрочном выходе.
    - Автоматическое обновление XP на Dashboard через инвалидацию провайдеров.

## 5. Спецификации данных (База)
- `profiles`: id, username, total_xp.
- `branches`: id, name, slug, description (используется для динамического построения Dashboard).
- `achievements`: id, name, description.
- `user_achievements`: связь пользователей и наград.

## 6. Принципы разработки
- Чистота кода: все общие виджеты выносятся в `sie_core`.
- Дизайн: Темная корпоративная эстетика (Deep Blue/Grey, Cyan accents).
- ИИ-ориентированность: Код пишется с учетом легкого чтения агентами Claude/Gemini.
